## 🔄 Data Flow

```
User Interaction (UI)
        ▼
 ┌─────────────┐
 │  Component  │ (uses Generic pages)
 └──────┬──────┘
        │ dispatches action
        ▼
 ┌─────────────────┐
 │  useModuleManager│ (composable)
 └──────┬──────────┘
        │ calls store action
        ▼
 ┌──────────────┐
 │ Pinia Store  │ (generated by factory)
 └──────┬───────┘
        │ makes API request
        ▼
 ┌──────────────┐
 │ HTTP Helper  │ (with caching)
 └──────┬───────┘
        │ sends request
        ▼
 ┌──────────────┐
 │ Laravel API  │
 └──────┬───────┘
        │ returns data
        ▼
   (Store updates state)
        │
        ▼
 (Component reactively updates)
```

## ✅ Checklist: Things to Know

- [ ] Module config is the source of truth
- [ ] Store is generated from config (don't write it!)
- [ ] Pages are generic (work with ANY config)
- [ ] Composables provide all operations
- [ ] Registry maps all modules centrally
- [ ] Helpers are shared across modules
- [ ] HTTP caching improves performance
- [ ] Full TypeScript support throughout
